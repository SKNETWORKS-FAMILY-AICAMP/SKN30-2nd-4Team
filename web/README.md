# 🎬 KO-BOX PREDICT 2026: 개봉작 흥행 예측 & 스크린 시뮬레이터 대시보드

이 모듈은 과거 8년간의 상영작 메타데이터와 사전 검색 버즈 트렌드를 학습한 **2-Stage 인공지능 앙상블 모델**을 활용하여, 아직 상영되지 않은 2026년 개봉 예정작(또는 최신 개봉작)의 **콘텐츠 흥행 잠재력**을 진단하고, **개봉 첫날 배급 스케일 확보량**에 따른 최종 예상 누적 관객수를 실시간 시뮬레이션하는 의사결정 대시보드입니다.

## 📂 1. 대시보드 파일 구성 및 아키텍처 (패키지 구조)

유지보수성 극대화 및 관심사 분리(SoC, Separation of Concerns)를 위해 정식 파이썬 패키지 구조로 설계되었습니다. 모든 모듈 경로 해석은 루트를 기준으로 무결하게 작동합니다.

```
web/
├── __init__.py                  # web 폴더를 파이썬 패키지로 선언
├── app.py                       # [Orchestrator] UI 그리드 및 Plotly 시각화 레이아웃 선언 (Thin Controller)
├── README.md                    # 대시보드 사용자 가이드 (이 문서)
│
├── utils/
│   ├── __init__.py              # utils 폴더를 파이썬 패키지로 선언
│   ├── loader.py                # [Data Layer] ML 모델, 2026 피처 데이터, 가드레일 캐싱 로드 및 필터링
│   ├── predictor.py             # [Domain Logic Layer] 2-Stage 추론 연산, 가중 기여도 계산, Elbow Point 추천 알고리즘
│   └── styles.py                # [Presentation Layer] Premium CSS 스타일 토큰 및 28개 피처 한글화 사전
│
├── feature_table_2026.csv       # 시뮬레이션 대상인 2026년 개봉 예정작 공통 피처 테이블 (v3 사양)
├── genre_guardrails.json        # 외삽 방지용 과거 장르별 배급 상하한선 가이드 가드레일 백업 데이터
├── stage1_ensemble.pkl          # Stage 1: CatBoost + XGBoost + MLP Stacking 흥행 잠재력 앙상블 모델
└── stage2_simulator.pkl          # Stage 2: 스크린/상영수 보정을 위한 단조 제약(Monotonic Constraint) XGBoost 모델
```

## 🖥️ 2. 대시보드 화면 구성

대시보드는 의사결정권자가 최적의 스크린 수 확보 전략을 직관적으로 도출할 수 있도록 **행 단위 통합 카드(Unified Row Containers) 레이아웃**으로 구성되어 있습니다. 별도의 사이드바 레이아웃 대신 모든 정보가 메인 영역 내에서 일관성 있게 조화됩니다.

```
+-----------------------------------------------------------------------------------+
| 🎬 2026 개봉작 흥행 예측 & 스크린 시뮬레이터                                              |
|                                                                                   |
| +-------------------------------------------------------------------------------+ |
| | [Row 1 통합 카드]                                                              | |
| | 🎬 영화 기본 정보 & 실제 배급 (좌)        | 📊 관객수 예측 요약 (우)                  | |
| |  - 개봉작 선택 셀렉트박스                  |  - 4개 핵심 예측/실적 메트릭              | |
| |  - 가로 2열 영화 기본 제원 및 실제값       |    (시뮬레이션 예상, 순수 콘텐츠 잠재력,  | |
| |  - 28개 입력 피처 데이터 상세 (Expander)   |     실제 최종 관객수, 배급 조건 오차율)    | |
| +-------------------------------------------------------------------------------+ |
| | [Row 2 통합 카드]                                                              | |
| | 💡 What-If 배급 스케일 시뮬레이터 (좌)     | 📈 배급 스케일 반응 곡선 (우)             | |
| |  - 가드레일 정보창 및 이중 슬라이더        |  - 스크린 수(X) 대비 최종 관객수(Y)        | |
| |  - 물리적 제약 비율 자동 검증기            |    Plotly 비선형 반응 곡선 차트           | |
| +-------------------------------------------------------------------------------+ |
|                                                                                   |
| +-------------------------------------------------------------------------------+ |
| | [하단 상세 분석 영역]                                                           | |
| | 📊 스크린 규모별 시나리오 테이블           | 💡 최적 배급 효율 추천 (Elbow Point)       | |
| | 📊 실제값 vs 예측값 바 차트               | 📊 2-Stage Stacking 모델 변수 중요도        | |
| +-------------------------------------------------------------------------------+ |
+-----------------------------------------------------------------------------------+
```

### ① Row 1 통합 카드 (영화 기본 정보 & 관객수 예측 요약)
*   **[좌측 열] 🎬 영화 기본 정보 & 실제 배급**:
    *   **2026년 개봉작 선택**: 분석 대상인 2026년 영화들을 고유 인덱스를 기반으로 선택할 수 있는 셀렉트박스가 메인 최상단에 배치됩니다.
    *   **기본 정보 & 실제 배급 가로 2열 배치**: 제목, 장르, 국가, 개봉일, 상영시간, 실제 첫날 스크린 및 상영 횟수가 깔끔한 2열 마크다운 그리드로 나열됩니다.
    *   **28개 피처 상세**: 모델이 의사결정을 내릴 때 사용한 28개 전처리 피처(감독/배우 스타파워, 사전 트렌드 버즈, 시즌 경쟁작 등)의 실제 수치를 한글화 매핑 사전을 통해 투명하게 제공합니다.
*   **[우측 열] 📊 관객수 예측 요약**:
    *   **2x2 메트릭 구조**: 시뮬레이션 예상 최종 관객수(실시간 예측치), 순수 콘텐츠 잠재 흥행력(Stage 1 점수 기준 순수 흥행력), 실제 최종 관객수, 실제 배급 조건 대입 시 오차율을 이중 가로 2열 레이아웃으로 균형 있게 표시합니다.

### ② Row 2 통합 카드 (What-If 시뮬레이터 & 반응 곡선)
*   **[좌측 열] 💡 What-If 배급 스케일 시뮬레이터**:
    *   **체급/장르 기반 이중 가드레일 슬라이더**: 기개봉작은 실제 배급 규모 대비 **30% ~ 200% 신뢰 영역**으로, 미개봉 신작은 과거 해당 장르의 상위 95% 분위 배포 통계치를 기준으로 작동하는 **장르 가드레일**이 자동 적용됩니다.
    *   **물리적 제약조건 (회차율) 검증기**: 슬라이더로 조절한 첫날 `상영 횟수 / 스크린 수` 비율을 실시간 연산하여 스크린당 평균 회차율을 검증하고 경고/안내 메시지를 동적으로 표시합니다.
*   **[우측 열] 📈 배급 스케일 반응 곡선**:
    *   첫날 스크린 확보량(X축)에 따른 예상 누적 관객수(Y축)의 비선형 변화 양상을 Area fill 스타일의 Plotly 인터랙티브 차트로 출력합니다. 현재 슬라이더 조절점과 실제 배급 예측점이 곡선 위에 마킹됩니다.

### ③ 하단 상세 분석 영역 (시나리오 테이블, 최적 추천, 성능 검증)
*   **📊 5대 구간 배급 시나리오 테이블**: 스크린 구간별 예측 최종 관객수와 이전 구간 대비 스크린 1개 추가 시 유입되는 **"한계 효율"**을 계산하여 테이블로 시각화합니다.
*   **💡 최적 배급 효율 추천 (Elbow Point)**: 최대 관객 잠재력의 **90% 이상을 안정적으로 수확할 수 있는 최소 스크린 확보 임계점(Elbow Point)**을 30단계 초정밀 스윕 알고리즘을 통해 탐색하여 추천 구간을 제안합니다.
*   **📊 상세 분석 그래프**: 실제 관객 실적과 모델의 다양한 예측 시나리오를 대조하는 px.bar 차트 및 2-Stage Stacking 모델 가중 중요도 Top 10 차트를 병렬로 제공합니다.

## 🔄 3. 구현 및 데이터 흐름 (Inference & Simulation Pipeline)

```
[2026 개봉 예정작/최신작]
      │
      ├── (Meta Features) ──────> [Stage 1: Stacking Ensemble]
      │                              │
      │                              ├──> pred_potential (콘텐츠 흥행 잠재력)
      │                              │       │
      │                              │       └──> expm1 역변환 ──> [✨ 순수 콘텐츠 잠재력]
      │                              ▼
      └── (What-If Control) ───> [Stage 2: Simulator (XGBoost)]
           - scrnCnt_day1            │ (With Monotonic Constraints)
           - showCnt_day1            ▼
                                 [최종 예상 누적 관객수] ──> [📈 반응 곡선 / 💡 Elbow Point 추천]
```

1.  **데이터 및 자원 로딩 (Loader)**: `app.py` 구동 시 `utils/loader.py`에 선언된 캐시 엔진이 발동하여 `.pkl` 바이너리 모델들과 `feature_table_2026.csv`를 메모리에 신속하게 적재합니다.
2.  **Stage 1: 순수 흥행력 진단 (Potential Predictor)**:
    *   선택된 영화의 장르, 과거 감독/배우 누적 흥행력, 극성수기 여부, 런타임, 경쟁 신작 수, 그리고 개봉 전 7일/30일 검색 버즈 데이터를 28개 입력 변수로 가공합니다.
    *   CatBoost, XGBoost, MLP 메타 앙상블 모델(`utils/predictor.py` 내 `compute_potential`)을 관통하여 배급 요인이 완전히 배제된 순수한 영화 흥행 등급인 **`pred_potential`**을 산출합니다.
3.  **Stage 2: 스크린 배급 스케일 시뮬레이션 (Scale Adjuster)**:
    *   사용자가 메인 영역의 실제 스펙 대비 이중 가드레일 한도 내에서 스크린 수와 상영 횟수를 슬라이더로 변경합니다.
    *   Stage 1에서 도출한 고유 흥행 점수 `pred_potential`에 슬라이더 조절 스크린/상영 횟수(Log)를 합성하여 `stage2_simulator.pkl` 모델에 주입합니다.
    *   단조 제약(Monotonic Constraint)이 적용된 XGBoost 회귀 모델을 통과한 뒤 최종 누적 관객수가 예측 산출되어 모든 컴포넌트(메트릭, Plotly 차트, 테이블, 시나리오 카드)로 즉시 전파되어 동적 렌더링을 갱신합니다.
