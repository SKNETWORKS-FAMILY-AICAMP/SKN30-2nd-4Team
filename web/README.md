# 🎬 KO-BOX PREDICT 2026: 개봉작 흥행 예측 & 스크린 시뮬레이터 대시보드

이 모듈은 과거 8년간의 상영작 메타데이터와 사전 검색 버즈 트렌드를 학습한 **2-Stage 인공지능 앙상블 모델**을 활용하여, 아직 상영되지 않은 2026년 개봉 예정작(또는 최신 개봉작)의 **콘텐츠 흥행 잠재력**을 진단하고, **개봉 첫날 배급 스케일 확보량**에 따른 최종 예상 누적 관객수를 실시간 시뮬레이션하는 의사결정 대시보드입니다.

## 📂 1. 대시보드 파일 구성 및 아키텍처 (패키지 구조)

유지보수성 극대화 및 관심사 분리(SoC, Separation of Concerns)를 위해 정식 파이썬 패키지 구조로 설계되었습니다. 모든 모듈 경로 해석은 루트를 기준으로 무결하게 작동합니다.

```

│
├── utils/
│   ├── __init__.py              # utils 폴더를 파이썬 패키지로 선언
│   ├── loader.py                # [Data Layer] ML 모델, 2026 피처 데이터, 가드레일 캐싱 로드 및 필터링
│   ├── predictor.py             # [Domain Logic Layer] 2-Stage 추론 연산, 가중 기여도 계산, Elbow Point 추천 알고리즘

├── genre_guardrails.json        # 외삽 방지용 과거 장르별 배급 상하한선 가이드 가드레일 백업 데이터
├── stage1_ensemble.pkl          # Stage 1: CatBoost + XGBoost + MLP Stacking 흥행 잠재력 앙상블 모델
└── stage2_simulator.pkl          # Stage 2: 스크린/상영수 보정을 위한 단조 제약(Monotonic Constraint) XGBoost 모델
```

### ① 좌측 사이드바 (Sidebar)
*   **2026년 개봉작 선택**: 2026년 5월 20일 이전 개봉한 분석 타겟 영화들을 선택합니다.
*   **영화 기본 정보**: 제목, 장르, 국가, 개봉일, 상영시간을 직관적으로 출력합니다.
*   **실제 배급 수치**: 해당 영화가 실제로 첫날 확보했던 스크린 수와 상영 횟수를 대조용 가이드라인으로 표시합니다.
*   **28개 피처 상세**: 모델이 의사결정을 내릴 때 사용한 28개 전처리 피처(감독/배우 스타파워, 사전 트렌드 버즈, 시즌 경쟁작 등)의 실제 수치를 한글 번역 사전(`styles.py`)을 통해 투명하게 조회합니다.

### ② 메인 1구역: 관객수 요약 카드 & What-If 시뮬레이터
*   **4열 요약 메트릭 카드**:
    1.  `시뮬레이션 예상 최종 관객수`: 사용자가 상영 조건을 조절함에 따라 실시간으로 변하는 결과 관객수
    2.  `✨ 순수 콘텐츠 잠재 흥행력`: 배급/스크린 요인을 완전히 제거하고 콘텐츠의 순수 체급으로 동원할 수 있는 **Stage 1 역변환 예측치**
    3.  `실제 최종 관객수`: KOBIS에 집계된 해당 영화의 실제 흥행 성적 (미개봉 신작은 "데이터 없음"으로 자동 예외 분기)
    4.  `실제 배급 조건 오차율`: 실제 스크린 설정값 입력 시 모델의 오차율을 제공하여 신뢰도 검증
*   **체급/장르 기반 이중 가드레일 슬라이더**:
    *   기개봉작은 실제 배급 규모 대비 **30% ~ 200% 신뢰 영역** 안에서만 슬라이더 조절 범위를 엄격하게 제한하여 기계학습 모델의 외삽 오류(Extrapolation Error)를 방지합니다.
    *   실제 수치가 적재되지 않은 미개봉 신작의 경우, 과거 해당 장르의 상위 95% 분위 배포 통계치를 기준으로 작동하는 **장르 기반 가드레일**이 자동으로 활성화됩니다.
*   **물리적 제약조건 (회차율) 검증기**:
    *   슬라이더로 조절한 첫날 `상영 횟수 / 스크린 수` 비율을 연산하여 **스크린당 평균 회차율**을 계산합니다.
    *   회차율이 1.0회 미만인 경우 **배급 경고**, 하루 상영 한계 시간(약 18시간)을 초과하는 8.0회 이상일 경우 **물리적 초과 경고**를 동적으로 경보하여 현실적인 배급 계획을 수립하도록 돕습니다.

### ③ 메인 2구역: 반응 곡선 및 최적 배급 효율 추천 (Elbow Point)
*   **📈 관객수 반응 곡선 Chart**: 첫날 스크린 확보량(X축)에 따른 예상 누적 관객수(Y축)의 비선형 변화 양상을 아름다운 Area fill 스타일의 Plotly 인터랙티브 차트로 출력합니다. 현재 슬라이더 조절점과 실제 배급 예측점이 곡선 위에 오버레이 마킹됩니다.
*   **📊 5대 구간 배급 시나리오 테이블**: 최저/최대 구간을 5개 영역으로 등분하여 대표 관객 시나리오를 데이터프레임으로 표현합니다. 특히 이전 구간 대비 스크린 1개 추가 시 유입되는 **"한계 효율 (추가 관객/스크린)"**을 수학적으로 자동 연산해 주어, 투자 효율 한계를 정확하게 짚어줍니다.
*   **💡 최적 효율 배급 추천**: 최대 수확 관객량의 **90%를 효율적으로 달성할 수 있는 최소 스크린 확보 임계점(Elbow Point)**을 정밀 30단계 스윕 알고리즘을 통해 계산하고, 최적의 스윗스팟(Sweet Spot) 구간과 적정 상영 횟수를 비즈니스 코멘트와 함께 제시합니다.

### ④ 메인 3구역: 상세 성능 비교 및 피처 중요도
*   **📊 실제값 vs 예측값 비교**: 영화 자체의 순수 체급, 실제 배급 조건, 시뮬레이터 조절값, 실제 관객 실적(0명 초과 시에만 출력되도록 자동 필터링 적용)의 체급 기둥들을 한눈에 대조 시각화합니다.
*   **📊 2-Stage Stacking 피처 기여도**: CatBoost 80% + XGBoost 20% Stacking 앙상블의 가중 피처 중요도 Top 10을 한글화하여 대시보드에 뿌려줌으로써, 콘텐츠 성공을 예측하는 데 가장 결정적이었던 요인(사전 검색 버즈 등)을 투명하게 증명합니다.

## 🔄 3. 구현 및 데이터 흐름 (Inference & Simulation Pipeline)

```
[2026 개봉 예정작/최신작]
      │
      ├── (Meta Features) ──────> [Stage 1: Stacking Ensemble]
      │                              │
      │                              ├──> pred_potential (콘텐츠 흥행 잠재력)
      │                              │       │
      │                              │       └──> expm1 역변환 ──> [✨ 순수 콘텐츠 잠재력]
      │                              ▼
      └── (What-If Control) ───> [Stage 2: Simulator (XGBoost)]
           - scrnCnt_day1            │ (With Monotonic Constraints)
           - showCnt_day1            ▼
                                 [최종 예상 누적 관객수] ──> [📈 반응 곡선 / 💡 Elbow Point 추천]
```

1.  **데이터 및 자원 로딩 (Loader)**: `app.py` 구동 시 `utils/loader.py`에 선언된 캐시 엔진이 발동하여 `.pkl` 바이너리 모델들과 `feature_table_2026.csv`를 메모리에 신속하게 적재합니다.
2.  **Stage 1: 순수 흥행력 진단 (Potential Predictor)**:
    *   선택된 영화의 장르, 과거 감독/배우 누적 흥행력, 극성수기 여부, 런타임, 경쟁 신작 수, 그리고 개봉 전 7일/30일 검색 버즈 데이터를 28개 입력 변수로 가공합니다.
    *   CatBoost, XGBoost, MLP 메타 앙상블 모델(`utils/predictor.py` 내 `compute_potential`)을 관통하여 배급 요인이 완전히 배제된 순수한 영화 흥행 등급인 **`pred_potential`**을 산출합니다.
3.  **Stage 2: 스크린 배급 스케일 시뮬레이션 (Scale Adjuster)**:
    *   사용자가 사이드바의 실제 스펙 대비 이중 가드레일 한도 내에서 스크린 수와 상영 횟수를 슬라이더로 변경합니다.
    *   Stage 1에서 도출한 고유 흥행 점수 `pred_potential`에 슬라이더 조절 스크린/상영 횟수(Log)를 합성하여 `stage2_simulator.pkl` 모델에 주입합니다.
    *   단조 제약(Monotonic Constraint)이 적용된 XGBoost 회귀 모델을 통과한 뒤 최종 누적 관객수가 예측 산출되어 모든 컴포넌트(메트릭, Plotly 차트, 테이블, 시나리오 카드)로 즉시 전파되어 동적 렌더링을 갱신합니다.
